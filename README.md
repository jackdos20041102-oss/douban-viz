## 一、可视化方案旨在解答的问题

豆瓣电影 Top 250 是国内使用最广泛的公众口碑榜单之一，但它通常以线性排名列表的形式呈现，用户很难从中发现结构性的规律。本项目将 250 条记录转化为一个多视图的交互探索工具，试图回答以下几个具体问题。

**类型与地域的分布规律**：哪些电影类型在高分榜单中占据主导位置？剧情、爱情、动画、犯罪这些类型各自的比重如何？从产出国角度看，美国、日本、中国香港、欧洲各国在 Top 250 中的贡献量与平均质量是否一致——即数量多的国家，均分是否也更高？

**评分与热度的关联**：高评分是否等同于高热度？评价人数（热度的代理指标）与最终评分之间的关系是线性的、还是存在明显的分层结构？是否存在一批"小众佳作"——评分极高但评价人数相对较少？

**时间维度的演变**：不同年代进入 Top 250 的影片数量与类型构成是否有显著差异？1990 年代与 2000 年代的上榜影片在平均评分上孰高孰低？哪个年代的影片在今天仍能获得最高的口碑认可？

**创作者维度的聚合**：哪些导演凭借多部作品同时进入榜单？他们的作品在类型和风格上是否具有一致性？

这四个问题并不是孤立的——通过多视图联动，用户可以在一次探索中同时观察类型、地域、年代与创作者之间的交叉关系，例如"日本动画在哪些年代最集中"或"克里斯托弗·诺兰的哪部作品热度最高"。

## 二、设计决策的依据

### 数据编码策略

**散点图（评分 × 评价人数）** 是整个工具的核心视图，承担着揭示两个最重要的连续变量之间关系的任务。评分映射至 X 轴，评价人数映射至 Y 轴并采用对数尺度——之所以用对数而非线性，是因为 Top 250 中评价人数的跨度极大（最低约 30 万，最高超过 350 万），线性尺度会让绝大多数点堆积在图表下方，失去区分度。点的半径同样编码评价人数（平方根缩放），颜色维度则允许用户在类型、国家、年代三种语义中切换，以便从不同角度观察聚类结构。

**环形图（类型占比）** 选用甜甜圈形而非传统饼图，原因在于人类对弧长的比较比对扇形面积的比较更准确，且中央空白区域可以放置总量数字，增加信息密度而不增加视觉噪声。图表右侧配合条形图例，将类型名称、绝对数量和占比百分比并列展示，使用户无需悬停即可读取精确数值，解决了传统饼图在类别数量较多时标签拥挤的问题。

**水平条形图（产出国分布）** 之所以选水平方向而非垂直，是因为国家名称是较长的中文字符串，垂直条形图中标签必须倾斜或截断，水平布局则可以将标签完整左对齐。该图支持在"数量"与"均分"两种指标之间切换，因为这两个维度指向不同的叙事——前者描述谁产出了最多的高分作品，后者描述谁的口碑最稳定，两者的排序并不相同，对比本身就是一个值得探索的发现。

**堆叠条形图（年代-类型分布）** 将时间维度与类型维度叠加。选择堆叠而非分组条形图，是为了同时保留两层信息：每个年代的影片总量（通过条形总高度）和各类型的内部构成（通过分段颜色）。如果使用分组，类别数量（6 种主要类型）会让每组过于拥挤，且年代间总量的对比会被分散。

**面积+折线图（年代均分走势）** 用折线追踪各年代的平均评分趋势，同时用淡色面积区域叠加影片数量的分布（Y 轴独立缩放），让用户在单张图中同时读出"哪个年代评分最高"和"哪个年代入选数量最多"，气泡大小进一步用面积编码数量，形成三重编码的验证。

**力导向图（导演-影片关系网络）** 适合表示非层次的关联关系。相比树状图或矩阵，力导向图能在不预设结构的情况下让数据自组织——导演与影片之间的连接数量差异会在空间布局中自然体现：多产导演因节点度数高而居于更中心的位置。图中导演节点以蓝色系区分，影片节点以类型颜色区分，支持拖拽和滚轮缩放，方便在节点密集区域进行局部探查。

### 替代方案的取舍

曾考虑使用地图呈现产出国分布，但豆瓣 Top 250 的来源极度集中（美国、日本、中国香港合计占比超过 70%），地图上大片地区为空白，信息密度过低，且地图本身隐含了地理相邻性的语义，与本数据无关，因此改用条形图。

曾考虑对散点图中的点使用透明度叠加来处理视觉遮挡问题，最终选择了hover时其他点透明度降低（0.12）的方案，这样在正常状态下所有点清晰可见，交互状态下焦点明确，比静态透明度更灵活。

影片详情面板（右侧抽屉）选择在点击后弹出而非常驻，避免占用大量屏幕空间影响图表的可用宽度，同时保持了"需要时才显示详情"的探索节奏。

## 四、开发流程概述

### 团队构成

本项目为独立完成的个人作业，无团队分工。

### 开发阶段

**数据获取阶段**

初期评估了多个候选数据集，包括城市空气质量指数（AQI）和电商销售数据，最终选择豆瓣 Top 250 的主要原因是：数据结构清晰（评分、类型、年份、地域等维度天然适合多视图分析）、数据量适中（250 条，不需要复杂的采样策略）、公众熟悉度高（便于用户对可视化结果进行直觉验证）。编写 Python 爬虫脚本（`1_scrape_douban.py`）进行采集，主要工作在于豆瓣页面的 HTML 结构解析和各字段的正则提取，尤其是评价人数和一句话短评的定位。

**架构搭建阶段**

使用 Vite 初始化 Vue 3 项目，配置 Tailwind CSS v4 和路径别名。设计 Pinia store 的数据结构，确定 `filtered`、`allGenres`、`allCountries`、`allDecades`、`directorNetwork` 等核心计算属性，使多视图联动在状态层而非组件层完成，避免跨组件的 props 传递。

**图表开发阶段**

按复杂度由低到高依次开发：统计卡片 → 评分直方图 → 国家条形图 → 环形图 → 散点图 → 年代堆叠图 → 年代趋势图 → 导演网络图。散点图的难点在于 brush 框选与点击事件的并存处理（两者共享同一 SVG 层级），以及对数尺度下的坐标轴刻度格式。网络图的难点在于力导向模拟的参数调优（排斥力、连接距离、碰撞半径）以及 zoom 与 drag 的事件层级隔离。

**交互与联动阶段**

实现全局搜索框与三个下拉筛选（类型、国家、年代）的联动逻辑，通过 Pinia store 的计算属性 `filtered` 将过滤结果广播至所有图表。实现点击图表元素设置高亮过滤（`highlightGenre`、`highlightCountry`、`highlightDecade`），点击数据点打开影片详情侧边栏，顶部导航通过 `IntersectionObserver` 实现滚动锚点联动。

**样式与调试阶段**

主要挑战是响应式图表的 `ResizeObserver` 问题：初始实现中图表容器使用 CSS `flex: 1` 自适应高度，`ResizeObserver` 在滚动容器内会将高度增长也视为尺寸变化并触发重绘，新绘制的 SVG 又撑大容器，形成正反馈循环，导致图表持续增高。最终通过为所有图表容器改用固定 `height` 值、并在回调中只响应宽度变化来解决。

## 五、可视化功能说明

应用分为四个导航区块，可通过顶部导航栏快速跳转：

**数据概览**：6 个统计卡片（收录影片数、平均评分、类型数、国家数、时间跨度、最高评分），评分-热度散点图，类型占比环形图。

**维度分析**：顶部类型快速筛选标签栏，产出国分布条形图（支持数量/均分切换），评分区间直方图，年代-类型堆叠图，年代均分走势图。

**关系网络**：导演-影片力导向关系图（仅展示有 2 部及以上作品入选的导演）。

**完整排行**：250 条影片的可搜索、多维度排序明细表。

全局交互：顶部搜索框（实时过滤片名和导演）、类型/国家/年代三个下拉筛选，所有视图实时联动。点击任意影片进入详情侧面板。

## 七、技术栈

- **Vue 3**（Composition API + `<script setup>`）— 组件化视图层
- **Pinia** — 全局状态管理，驱动多视图联动
- **D3.js v7** — 全部图表绘制（不依赖封装图表库）
- **Tailwind CSS v4** — 样式工具类
- **Vite 8** — 构建工具
- **Python 3**（requests + BeautifulSoup4）— 数据爬取
